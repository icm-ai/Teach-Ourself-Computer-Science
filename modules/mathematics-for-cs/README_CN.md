# 计算机科学中的数学

[English](README.md) | 简体中文

## 概述

离散数学基础，包括逻辑、组合数学、概率和图论。本模块为计算机科学的理论研究提供必要的数学工具。

## 为何要学？

计算机科学基本上是应用数学的一个「跑偏的」分支，因此学习数学将会给你带来竞争优势。

从某个角度说，计算机科学是应用数学的一个「发育过度」的分支。尽管许多软件工程师试图——并且在不同程度上成功做到——忽视这一点，我们鼓励你用学习来拥抱数学。如若成功，比起那些没有掌握数学的人，你将获得巨大的竞争优势。

> 如果人们不相信数学是简单的,那么只能是因为他们没有意识到生活有多么复杂。
>
> — John von Neumann

## 前置要求

**推荐：**

- 基础数学 - 高中代数和基本逻辑推理

## 数学主题

对于计算机科学，数学中最相关的领域是「离散数学」，其中的「离散」与「连续」相对立，大致上指的是应用数学中那些有趣的主题，而不是微积分之类的。

**核心主题：**

- 逻辑和证明
- 集合论
- 排列组合
- 概率论
- 图论
- 数论基础（密码学相关）

**补充主题：**

- 线性代数（计算机图形学和机器学习）

## 练习

### 教材习题

**来源：** 《计算机科学中的数学》各章节习题

**难度：** 初级到高级

**主题：** 证明、组合、概率、图论

**建议：** 证明题非常重要，不要跳过。手写证明过程有助于理解。

### 编程实现

**实践项目：**

- 实现图算法（DFS、BFS、最短路径）
- 编写组合生成器（排列、组合）
- 实现简单的密码算法（基于数论）
- 模拟概率问题（蒙特卡洛方法）

## 学习方法

### 推荐学习路径

1. **入门阶段**
   - 观看 3Blue1Brown 的线性代数系列（可选但推荐）
   - 阅读 Lovász 的讲义了解离散数学

2. **核心学习**
   - 学习《计算机科学中的数学》
   - 观看 MIT 6.042J 视频课程
   - 完成每章的习题

3. **线性代数**（如果需要）
   - 观看 3Blue1Brown 的 Essence of Linear Algebra
   - 学习 Gilbert Strang 的《线性代数导论》
   - 观看 MIT 18.06 课程

4. **应用巩固**
   - 将数学概念应用到算法学习中
   - 实现数学算法
   - 解决实际问题

### 时间分配建议

- **离散数学：** 100-150 小时
- **线性代数：** 50-100 小时（如需要）
- **总计：** 150-250 小时

### 学习技巧

1. **手写证明** - 数学最好在纸上学习，不是屏幕上
2. **理解直觉** - 不要只记公式，要理解背后的含义
3. **连接应用** - 思考数学概念如何应用到计算机科学
4. **循序渐进** - 数学需要时间积累，不要急于求成
5. **多做练习** - 数学能力来自大量练习

## 学习次序建议

离散数学和算法的关系为例：先学习数学可以帮助你更深入地分析和理解算法，然而先学习算法可以为学习离散数学提供更大的动力和应用背景。

**建议：** 可以与算法与数据结构并行学习，相互促进。

## 常见问题

### 我必须学数学吗？

不是必须，但强烈推荐。数学帮助你：

- 更深入地理解算法
- 分析算法复杂度
- 理解密码学和安全
- 在机器学习和图形学方面更有竞争力

### 线性代数重要吗？

取决于你的方向：

- **必须学** 如果你要做机器学习、计算机图形学、计算机视觉
- **推荐学** 对所有计算机科学家都有益
- **可以缓** 如果你主要做 Web 开发或应用开发

### 如何保持数学不生疏？

- 在学习其他主题时有意识地应用数学
- 定期复习关键概念
- 解决需要数学思维的编程问题
