# 算法与数据结构

[English](README.md) | 简体中文

## 概述

基本算法、数据结构和复杂度分析，用于高效解决问题。本模块涵盖常见的数据结构（数组、链表、栈、队列、树、图等）以及经典算法（排序、搜索、动态规划、图算法等）。

## 为何要学？

如果你不懂得如何使用栈、队列、树、图等常见数据结构，遇到有难度的问题时，你将束手无策。

正如几十年来的共识，计算机科学教育所赋予人们的最大能力在于对常见算法和数据结构的熟悉。此外，这也可以训练一个人对于各种问题的解决能力，有助于其他领域的学习。

## 推荐资源

### 教材

#### 《算法设计手册》

**作者：** Steven Skiena

**中文版：** [《算法设计手册》第2版](https://book.douban.com/subject/4048566/)

**英文版：** The Algorithm Design Manual

**难度：** 中级

**为什么选择这本书：**

显而易见，Skiena 对此充满热爱，迫不及待地想要帮助其他人理解。在我们看来，这本书给人一种焕然一新的体验，完全不同于那些更加经常被推荐的书（比如 Cormen、Leiserson、Rivest、Stein 的《算法导论》或 Sedgewick 的书，后两者充斥着过多的证明，不适合以**解决问题**为导向的学习）。

**主要内容：**

- 算法分析基础
- 数据结构（数组、链表、栈、队列、树、图、哈希表）
- 排序和搜索
- 图算法（遍历、最短路径、最小生成树）
- 组合搜索和启发式方法
- 动态规划
- NP完全性

#### 《算法导论》（可选，作为参考）

**作者：** Cormen, Leiserson, Rivest, Stein (CLRS)

**中文版：** [《算法导论》第3版](https://book.douban.com/subject/20432061/)

**难度：** 高级

**何时使用：** 作为参考书，查阅特定算法的严格证明和理论分析

## 练习

### 编程练习

#### LeetCode 题目

**来源：** [LeetCode](https://leetcode.com/) / [力扣](https://leetcode.cn/)

**难度：** 简单、中等、困难

**预计时间：** 每题 30分钟-2小时

**建议：** 我们建议解决大约 100 道随机挑选的 LeetCode 问题，作为学习的一部分。

**学习策略：**

1. **按主题学习** - 先集中练习某一类数据结构或算法
2. **循序渐进** - 从简单题开始，逐步提升难度
3. **多次尝试** - 重要的题目要多做几遍
4. **学习他人解法** - 看优秀的解答，学习不同的思路

**推荐题目类型：**

- 数组和字符串（20题）
- 链表（10题）
- 栈和队列（10题）
- 树和图（20题）
- 递归和回溯（10题）
- 动态规划（15题）
- 排序和搜索（10题）
- 其他经典问题（5题）

#### 《算法设计手册》练习

**来源：** 书中习题

**难度：** 中级

**主题：** 各章节对应的算法和数据结构

## 学习方法

### 推荐学习路径

1. **阅读《算法设计手册》**
   - 按章节顺序学习
   - 理解算法的设计思想，而不仅仅是实现

2. **观看视频课程**
   - 选择 Skiena 或 Roughgarden 的课程
   - 配合阅读观看，加深理解

3. **刷 LeetCode**
   - 按主题练习，每个主题至少10题
   - 先自己思考，实在想不出再看提示
   - 学习多种解法，理解时间和空间复杂度

4. **深入学习**（可选）
   - 参加算法竞赛（Codeforces、AtCoder）
   - 阅读《算法导论》深入理论
   - 研究高级主题（网络流、字符串算法等）

### 时间分配建议

- **最少投入：** 100 小时（基础数据结构和算法 + 50 题 LeetCode）
- **推荐投入：** 150-200 小时（完整学习 + 100 题 LeetCode）
- **深入学习：** 300+ 小时（高级算法 + 竞赛编程）

### 学习技巧

1. **理解优先** - 不要死记硬背，理解算法的核心思想
2. **画图辅助** - 许多算法在纸上画图更容易理解
3. **分析复杂度** - 每个算法都要分析时间和空间复杂度
4. **多种语言** - 尝试用不同语言实现同一算法
5. **解决问题** - 学习算法是为了解决实际问题，不是应付面试

## 常见问题

### 我应该先学算法还是先刷题？

两者结合效果最好：

1. 先学习一类数据结构/算法的理论
2. 立即做相关的练习题巩固
3. 遇到新题型再回去学理论

不要只看书不做题，也不要只刷题不学理论。

### LeetCode 要刷多少题？

- **找工作准备：** 150-300题（覆盖各种类型）
- **基础学习：** 100题左右（每种类型都练习到）
- **保持手感：** 每周2-3题

质量比数量重要。100道真正理解的题比500道囫囵吞枣的题更有价值。

### 需要学习《算法导论》吗？

不一定。《算法导论》更适合：

- 研究生课程
- 需要严格数学证明的研究
- 作为参考书查阅

对于大多数软件工程师，《算法设计手册》就足够了。

### 算法竞赛有必要参加吗？

看你的目标：

- **不必参加** 如果你只想掌握常用算法
- **推荐参加** 如果你想深入学习或进入顶级科技公司
- **量力而行** 竞赛编程需要大量时间投入
